# 学习指南

虚幻引擎（UE）涵盖了大量概念与工具，要一步步掌握这样庞大的知识体系，确实需要一套行之有效的学习方法——**就像探险时需要地图和指南针一样**。

## 学习方法论

其实学习虚幻引擎并不像想象中那么复杂，推荐参考视频 [快到不公平的学习法 - Bilibili](https://www.bilibili.com/video/BV1wT1FBmEP3)，我们将其核心理念提炼为以下五点：

1. **行动先行**：不要过度准备，直接动手实践。虚幻引擎本质上是一个工具，就像螺丝刀或锤子，只有在实际使用中才能真正掌握它，并在过程中发现并解决问题。
2. **二八法则**：通常20%的核心知识能带来80%的效果产出。找到那关键的20%，其余内容会在后续实践中自然补全。
3. **高强度的微练习**：每天坚持20-30分钟的专项高强度练习，将其培养成习惯。避免将任务拉长成马拉松，保持专注和效率。
4. **费曼学习法**：只有当你能够用自己的语言清晰解释所学内容时，才算真正理解。试着向他人讲解你学到的知识，通过回答问题来检验自己的理解程度。
5. **学以致用**：启动自己的小项目，发挥创造力，将学到的知识付诸实践。不要害怕犯错、失败或不完美，在学习进步的阶梯上，**完成远比完美更重要**。

这些方法你可能在其他地方也见过，请相信——**这确实是高效学习的捷径**！

## 虚幻引擎中的二八法则

你可能见过那些如同参天大树般庞大的虚幻引擎知识图谱，它们确实展示了虚幻引擎的深度和广度——**但对你而言，所有这些真的都那么重要吗？**

开发游戏最核心的要素可以归结为两点：**玩法与画面**。

- **玩法**：核心在于3C系统——角色（Character）、相机（Camera）和控制（Control）
  - 这是虚幻引擎的核心功能：让开发者能够定义玩家输入（键盘、鼠标、手柄等）如何影响游戏世界中的3C元素，从而为玩家提供掌控感和愉悦感
  - **实现方式**：通过编写逻辑代码（蓝图或C++）来调用引擎提供的丰富接口
- **画面**：作为开发者，你需要决定向玩家呈现什么样的视觉世界——是2D像素风格、3D写实场景，还是风格化的卡通渲染？
  - 虚幻引擎的另一核心功能：让开发者能够构建游戏场景，并控制玩家看到的内容和方式
  - **实现方式**：利用引擎提供的场景编辑工具，布置场景元素并设置其材质属性

游戏开发中80%的核心工作，基本上都围绕这两个方面展开。

所以，当你需要开始学习其中一项内容时，先问自己：**“这对于你做游戏有什么帮助？”**
